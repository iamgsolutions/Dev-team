# SKILL: Product quality — what makes software SELLABLE

This team builds software that MG Solutions SELLS. Every product
decision is measured against: would a customer pay for this exactly as it is?

- **Empty states**: every list/table defines what's shown with no data (a useful
  message + an action to create the first one). A blank screen looks broken.
- **Immediate feedback**: every user action produces a visible response
  in <1s (spinner, toast, transition). Silence looks like a freeze.
- **Errors in plain language**: never show stacktraces or "Error 500". A clear
  message + what the user can do. Keep the technical log separate, for us.
- **Realistic data in the demo**: seed coherent sample data
  (no "asdf", no "test123") — the first impression is what sells.
- **A polished minimum > a half-baked lot**: one perfect Must-have feature
  is worth more than three mediocre Should-haves. Cut scope, never quality.
